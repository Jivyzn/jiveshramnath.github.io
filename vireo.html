CASE STUDY / 01

Vision translated into action.

JØ7 Vireo is a low-cost, offline assistive-vision platform for blind and visually impaired users. It combines object detection, distance awareness, speech and haptic feedback on edge hardware.

Privacy by defaultOffline inferenceLow-cost designAccessible feedback
[01] SYSTEM ARCHITECTURE

Sense. Interpret. Communicate.

The system is designed so perception never ends as a bounding box on a screen. It must become useful guidance.

01Capture

Camera and distance sensors observe the near environment.

02Infer

Object detection runs locally on Raspberry Pi and Hailo acceleration.

03Prioritise

Software decides what is relevant enough to interrupt the user.

04Respond

Speech and a haptic belt communicate direction and urgency.

[02] DESIGN CONSTRAINTS

The constraints shaped the invention.

01
Affordability

Keep the full system below premium assistive-device pricing.

Commodity compute, repurposed cells and modular components reduce cost without forcing cloud dependence.

02
Privacy

Video should not need to leave the device.

Local inference protects environmental imagery and keeps the system functional when connectivity disappears.

03
Cognitive load

More detections can make the product worse.

Useful feedback requires prioritisation, timing and restraint rather than narrating every object in view.

04
Failure

Safety signals need redundancy.

Visual inference, distance sensing and haptics cover different weaknesses instead of pretending one sensor is infallible.

Vireo is a system, not a science-fair prop.
